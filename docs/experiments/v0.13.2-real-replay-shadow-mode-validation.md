# v0.13.2 real replay shadow-mode validation

## Purpose

v0.13.2 validates that the existing v0.13.0 config-driven offline shadow-mode controller runner can be executed against a real v0.8.0 replay dataset.

The stage uses the binding discovered in v0.13.1.

## Input dataset

The v0.13.1 binding selected:

    results/v0.8.0/runs/v0.8.0-M4-SINE-L80-A48-T720/sine-timeseries.csv

The required replay columns are:

- u_cmd as the command input column;
- u_ach as the achieved-output column.

The real replay file also contains time columns such as t_sec and t_rel_sec. v0.13.2 records these as source-time annotations.

## Relationship to v0.13.0

v0.13.2 does not replace the v0.13.0 runner and does not add a new controller.

Instead, it generates a resolved v0.13.0 runtime config that points to the real v0.8.0 replay dataset, runs the existing offline runner, and then writes an annotated output that preserves the real replay time column.

## Timebase handling

The inherited v0.13.0 runner writes its internal time_seconds field as a row index.

v0.13.2 therefore treats the runner time_seconds field as row-index time and separately records source_time_seconds from the real replay time column when available.

When a real time column is available, v0.13.2 derives a median positive delta and uses it as the runner sample_time_seconds in the resolved config. This is a configuration-level replay alignment step, not a new controller behavior.

## What is not run

This stage is offline-only.

It does not:

- call kubectl;
- modify Kubernetes;
- start a live controller;
- generate transaction load;
- apply controller recommendations;
- execute closed-loop control;
- use live telemetry.

## Expected decision

A successful run writes:

    REAL_REPLAY_SHADOW_MODE_VALIDATION_COMPLETE

This means that the offline shadow-mode runner has executed against the real v0.8.0 replay file.

It does not mean the controller is safe for live actuation.

## Reproduction

Run:

    python3 scripts/v0.13.2/run-real-replay-shadow-validation.py --config configs/v0.13.2/real-replay-shadow-validation.json

Expected outputs:

- results/v0.13.2/real-replay-shadow-mode-validation/runtime-shadow-controller-replay.json
- results/v0.13.2/real-replay-shadow-mode-validation/shadow-controller-timeseries.csv
- results/v0.13.2/real-replay-shadow-mode-validation/shadow-controller-summary.json
- results/v0.13.2/real-replay-shadow-mode-validation/real-replay-shadow-timeseries.csv
- results/v0.13.2/real-replay-shadow-mode-validation/real-replay-shadow-summary.json
