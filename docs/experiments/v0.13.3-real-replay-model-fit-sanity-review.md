# v0.13.3 real replay model-fit sanity review

## Purpose

v0.13.3 checks whether the plant approximation used by the offline shadow-mode runner is consistent with the real v0.8.0 replay output.

This stage separates two claims:

1. the replay pipeline works;
2. the model is scientifically good enough for controller claims.

v0.13.2 established the first claim. v0.13.3 reviews the second claim.

## Input

The review uses:

    results/v0.13.2/real-replay-shadow-mode-validation/real-replay-shadow-timeseries.csv

The primary columns are:

- u_cmd_replay as the replay command;
- u_ach_replay as the achieved replay output;
- y_hat as the model prediction;
- source_time_seconds as the preserved real replay time.

## Analysis

The script computes:

- MAE;
- RMSE;
- bias;
- prediction-error standard deviation;
- underestimation and overestimation fractions;
- correlation between y_hat and u_ach_replay;
- linear calibration from y_hat to u_ach_replay;
- early-transient metrics;
- later-replay metrics.

## Decision interpretation

MODEL_FIT_SANITY_REVIEW_COMPLETE means the current model passed the configured basic sanity thresholds.

MODEL_FIT_WEAK_BUT_PIPELINE_VALID means the replay pipeline is valid, but the model fit is too weak for controller-quality claims.

MODEL_FIT_SANITY_REVIEW_INSUFFICIENT_DATA means there were not enough valid prediction rows for the review.

## What is not run

This stage is offline-only.

It does not:

- call kubectl;
- modify Kubernetes;
- start a live controller;
- generate transaction load;
- apply controller recommendations;
- execute closed-loop control.

## Reproduction

Run:

    python3 scripts/v0.13.3/review-model-fit.py --config configs/v0.13.3/model-fit-sanity-review.json

Expected outputs:

- results/v0.13.3/model-fit-sanity-review/model-fit-summary.json
- results/v0.13.3/model-fit-sanity-review/model-fit-timeseries.csv
