# v0.13.4 real replay throughput model refit

## Purpose

v0.13.4 refits the discrete plant approximation used by the offline shadow-mode controller workflow.

The motivation is that v0.13.3 reviewed the placeholder model fit against real replay data and established the need for a more evidence-based plant approximation before making stronger controller-quality claims.

## Scope

This stage is offline-only.

It does not:

- call kubectl;
- modify Kubernetes;
- start a live controller;
- generate transaction load;
- apply controller recommendations;
- execute closed-loop control;
- use live telemetry.

## Input

The refit uses the v0.13.2 real replay shadow-mode timeseries:

    results/v0.13.2/real-replay-shadow-mode-validation/real-replay-shadow-timeseries.csv

The selected columns are:

- `source_time_seconds` as the time column;
- `u_cmd_replay` as the command input;
- `u_ach_replay` as the observed output.

## Method

The script fits a discrete first-order no-intercept model:

    y[k+1] = a_y * y[k] + b_u * u[k-delay]

The script evaluates multiple delay candidates and selects the candidate with the lowest one-step RMSE.

Candidate delays:

    0, 1, 2, 3, 4, 5, 10, 15, 20

## Selected model

The selected model is:

- `a_y = 1.02763034203239`
- `b_u = -0.03997702623417542`
- `delay_steps = 15`
- `y0 = 80.0`

## Results

The selected one-step fit produced:

- RMSE: `5.586271591418611`
- MAE: `3.110984215151178`
- bias: `-0.627255234576628`
- MAPE: `5.195506583982073`
- R2: `0.9730152858434464`

The stage decision is:

    REAL_REPLAY_MODEL_REFIT_COMPLETE

## Interpretation

The fitted coefficients provide a strong one-step-ahead approximation on the real replay series.

However, the model structure does not include an intercept or equilibrium-deviation term. Therefore, good one-step fit metrics are not sufficient to conclude that the model will behave well in recursive/free-run shadow-mode simulation.

The fitted coefficients must be evaluated in the v0.13.5 placeholder-vs-fitted shadow-mode comparison before being considered as a default model candidate for closed-loop preparation.
