# v0.14.0 offline controller simulator with selected recursive model

## Purpose

v0.14.0 introduces an offline controller simulator based on the v0.13.6 selected recursive model.

The simulator is a preparation tool for controller logic. It is not a live controller and does not apply actions to Kubernetes, a validator, or a transaction-load process.

## Methodological context

- v0.13.4 produced a one-step throughput model refit.
- v0.13.5 showed caution for placeholder vs one-step refit shadow comparison.
- v0.13.6 selected a recursive plant model with pass status.
- v0.13.7 confirmed the selected recursive model as candidate surrogate plant model.
- v0.14.0 uses that model inside an offline closed-loop simulation harness.

## Failure analysis

Detailed failure analysis is recorded in:

results/v0.14.0/offline-controller-simulator-selected-recursive-model/failure-analysis.md

The failure is concentrated in the multistep profile. At step 35 the target drops from 104.0 to 38.0. With max_step_change = 16.0, the simulated controller cannot instantaneously command the new lower target, so transient max absolute error exceeds fail_max_abs_error = 48.0.

This is treated as a valid simulator-gate failure, not as an implementation failure.
