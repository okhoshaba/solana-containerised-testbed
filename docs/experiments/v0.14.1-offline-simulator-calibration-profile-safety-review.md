# v0.14.1 offline simulator calibration and profile safety review

## Purpose

v0.14.1 reviews the v0.14.0 offline simulator fail result and determines whether the failures are hard simulator failures or expected transient behaviour caused by actuator rate limits.

## Methodological context

- v0.14.0 created the first offline closed-loop simulation harness.
- v0.14.0 failed because the multistep profile produced transient max absolute error above the configured fail threshold.
- v0.14.1 keeps that result intact and performs a separate calibration review.

## Calibration method

The review identifies target-change events, excludes a configured transient window after each target change for settled-window metrics, and checks whether maximum error occurred during a rate-limited target-change transient.
