# v0.14.3 selected PID baseline closed-loop validation

## Purpose

v0.14.3 formalises the selected controller from v0.14.2 as a fixed reference baseline before MPC work.

The purpose is not to optimise PID gains. The purpose is to establish a stable, reproducible, and documented baseline controller against which future MPC behaviour can be compared.

## Selected baseline

The selected controller is:

    p_kp0.350_ki0.000_kd0.000

Controller gains:

    kp = 0.35
    ki = 0.0
    kd = 0.0

Although this belongs to the PID-style controller research line, the selected controller is effectively a proportional-only controller. This is intentional: the calibrated v0.14.2 sweep selected zero integral and derivative terms.

## Why this stage is needed before MPC

MPC comparison requires a fixed reference controller. Without this stage, the project would risk comparing MPC against an unstable moving target: a PID family that continues to be retuned.

v0.14.3 prevents that problem by freezing the controller and validating it independently as a baseline.

## Difference from v0.14.2

v0.14.2 performed a gain sweep across P, PI, and PID candidates.

v0.14.3 validates exactly one candidate:

    p_kp0.350_ki0.000_kd0.000

Gain sweep and retuning are disabled.

## Decision

Status: pass

The fixed selected baseline has no unexplained failures, no settled warnings, and acceptable settled RMSE under calibrated offline closed-loop metrics.

## Interpretation

The result supports using the selected controller as an offline reference baseline before MPC experiments.

The conclusion is deliberately narrow. It does not claim that the selected controller is globally optimal, production-safe, or superior to future MPC logic.

## MPC comparison role

Future MPC stages should compare against this fixed P-only baseline and should report whether MPC improves, worsens, or trades off:

- settled error
- transient response
- overshoot and undershoot behaviour
- saturation and rate-limit behaviour
- safety classification
- recovery behaviour after target changes
