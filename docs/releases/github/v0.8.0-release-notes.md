# Solana Containerised Testbed v0.8.0: Dynamic Load System Identification

## Summary

This release adds the `v0.8.0 dynamic load system identification` stage to the Solana Containerised Testbed.

The release provides controlled dynamic-load experiments and supporting analysis scripts for initial throughput-channel system identification of the testbed.

The main identification channel is:

    u_cmd(t) -> u_ach(t)

where `u_cmd(t)` is the commanded transaction load and `u_ach(t)` is the achieved throughput.

## Experiment classes

| Class | Description | Status |
|---|---|---:|
| M0 | Steady-state baselines | PASS |
| M1 | Upward step-response experiments | PASS |
| M2 | Step-down and recovery experiments | PASS |
| M3 | Multi-step load profiles | PASS |
| M4 | Stepwise sine-approximation / initial frequency-response experiment | PASS |

Confirmed safe identification range:

    lambda = 32..128

## Main findings

- All M0-M4 experiment classes completed successfully.
- Observed `err_per_sec` remained zero across the included experiment classes.
- Throughput tracking is close to unity gain.
- Step-like transitions settle within approximately one 5-second sample interval.
- The M4 sine-approximation run shows near-unity low-frequency gain and very high correlation between commanded and achieved throughput.

Key M4 indicators:

- `gain_peak_to_peak = 1.002119`
- `gain_sinusoidal_fit = 0.998294`
- `zero_lag_correlation = 0.999539`
- `phase_lag_seconds = 1.223330`
- `RMSE = 0.997340 TPS`
- `MAE = 0.421263 TPS`

Overall conclusion:

    throughput_channel_ready_for_initial_modelling = true
    latency_channel_ready_for_modelling = false

## Known limitations

The current `collect_csv.csv` outputs do not contain usable `lat_p99` values.

Therefore:

- v0.8.0 supports throughput-channel system identification;
- v0.8.0 does not yet support latency-constrained MPC;
- latency-aware modelling requires fixing or extending telemetry collection.

Additional limitations:

- settling-time estimates are limited by the current 5-second sampling resolution;
- the M4 sine experiment is a stepwise sine approximation;
- the release is intentionally limited to `lambda=32..128`;
- `lambda=512` is excluded due to previously identified storage-pressure risk.
