# Solana Containerised Testbed v0.4.0: Dedicated KVM Multi-Node Kubernetes Deployment

## Release status

This release documents and validates the deployment of the Solana Kubernetes Observability Core on a dedicated KVM-based multi-node Kubernetes cluster.

Version `v0.4.0` extends the previous Minikube-based Kubernetes validation by deploying the same observability-oriented Solana workload on a dedicated kubeadm cluster composed of one control-plane virtual machine and two worker virtual machines.

## Summary

The release validates the following deployment path:

    CentOS Stream 9 host
      -> KVM/libvirt
      -> Ubuntu Server 24.04 virtual machines
      -> kubeadm-based Kubernetes cluster
      -> Solana Kubernetes Observability Core

The deployment remains focused on infrastructure and observability. It does not introduce controlled load generation, dashboarding, MPC, single-agent RL, MARL, or Agave.

## Main additions

This release adds and validates:

- dedicated KVM-based three-node Kubernetes topology;
- Ansible automation for node preparation, cluster bootstrap, validation, and reset support;
- containerd runtime on Ubuntu Server worker nodes;
- Calico CNI;
- local-path-provisioner as the default StorageClass;
- validator placement on `testbed-role=validator`;
- monitor placement on `testbed-role=observability`;
- explicit `local-path` storage for the validator ledger PVC;
- dedicated `k8s/overlays/kvm-multinode` Kustomize overlay;
- deployment evidence under `results/kvm-multinode-observability-core`;
- wallet transfer validation evidence.

## Deployed components

The deployed Solana Kubernetes Observability Core includes:

- `solana-observability` namespace;
- validator StatefulSet;
- validator ledger PersistentVolumeClaim;
- wallet-init Job;
- latency monitor Deployment;
- RPC Service on port `8899`;
- Yellowstone/Geyser gRPC Service on port `10000`;
- metrics Service on port `9464`;
- wallet transfer validation Job.

## Validation summary

The deployment was validated through:

- Kubernetes node readiness checks;
- workload placement checks;
- PVC binding checks;
- service and EndpointSlice checks;
- validator logs;
- monitor logs;
- wallet-init logs;
- RPC `getHealth` validation;
- metrics endpoint validation;
- wallet transfer validation.

The wallet transfer validation confirmed:

- Solana CLI version `1.18.25`;
- cluster version `1.18.25`;
- creation of a temporary sender keypair;
- airdrop of `2 SOL` to the sender;
- transfer of `0.5 SOL` to the receiver;
- confirmed transaction signature;
- sender balance update from `2 SOL` to `1.499995 SOL`;
- receiver balance update from `10000 SOL` to `10000.5 SOL`.

The metrics endpoint reported a transaction latency sample after the validation transfer:

    solana_transaction_latency_seconds_count 1

It includes playbooks for:

## Scope

This release belongs to the infrastructure and observability stage of the project.

The following remain out of scope:

- synthetic load generation;
- dashboard development;
- Prometheus/Grafana stack;
- MPC;
- single-agent RL;
- MARL;
- Agave.

Agave remains a separate research branch.
