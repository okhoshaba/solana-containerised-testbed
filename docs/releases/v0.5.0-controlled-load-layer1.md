# Solana Containerised Testbed v0.5.0-controlled-load-layer1

## Release purpose

This release prepares the Solana Containerised Testbed for Controlled Load Layer 1.

It extends the validated KVM-based multi-node Kubernetes observability baseline with controlled, reproducible transaction load generation.

The methodological sequence is:

infrastructure -> observability -> controlled load

This release does not introduce MPC, reinforcement learning, MARL, adaptive control policy learning, or Agave migration. Those remain later or separate research branches.

## Baseline

The release builds on the previously validated Kubernetes Observability Core baseline:

- dedicated KVM-based multi-node Kubernetes cluster;
- kubeadm-based Kubernetes deployment;
- validator workload scheduled on testbed-role=validator;
- observability workload scheduled on testbed-role=observability;
- validator RPC Service on port 8899;
- metrics Service on port 9464;
- Yellowstone/Geyser gRPC Service on port 10000;
- wallet transfer validation completed;
- transaction latency telemetry observed.

## Added components

Controlled Load Layer 1 adds:

- loadgen2, a controlled Solana transaction load generator;
- CSV collection tooling for controlled-load experiments;
- knee-step and adaptive probe execution scripts;
- controlled-load dashboard code;
- container definitions for the load generator and tools image;
- local runtime wiring for smoke testing;
- Kubernetes manifests for the controlled-load layer;
- Kubernetes PVC-backed result persistence;
- validation notes for local and Kubernetes smoke runs;
- candidate run evidence for a reproducible multi-level Kubernetes experiment.

## Kubernetes resources

The controlled-load layer uses the existing solana-observability namespace and adds:

- solana-loadgen2 Deployment;
- solana-loadgen2 ClusterIP Service on port 7070;
- controlled-load-results-pvc;
- controlled-load-knee-step Job definition;
- manually created solana-payer-keypair runtime Secret.

The payer keypair is not stored in Git.

## Known limitations

This release provides controlled transaction load and observability integration, but it does not yet provide:

- a formal saturation model;
- automated knee-point detection as a validated research result;
- controller feedback logic;
- MPC-based control;
- reinforcement learning or multi-agent reinforcement learning;
- production-grade benchmarking claims.
