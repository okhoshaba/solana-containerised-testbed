## Description

This software release extends the Solana Containerised Testbed with Controlled Load Layer 1.

The release builds on a validated KVM-based multi-node Kubernetes observability baseline and introduces controlled, reproducible transaction load generation inside the same Kubernetes environment. The methodological sequence is:

infrastructure -> observability -> controlled load

The release adds the loadgen2 transaction load generator, controlled-load CSV collection tooling, execution scripts, container definitions, local smoke-test wiring, Kubernetes manifests, PVC-backed result persistence, and validation notes.

The Kubernetes controlled-load layer was validated using the existing solana-observability namespace, the in-cluster solana-rpc:8899 Service, the solana-metrics:9464 Service, and a manually created runtime Secret for the payer keypair. The payer keypair and Kubernetes Secret material are not included in the repository.

A candidate reproducible Kubernetes run was performed with load levels 1 2 4 8, 60 seconds per level, and 5 second sampling. The post-run state reported sent_total=1401, ok_total=1401, err_total=0, and an empty last_err.

This release does not introduce MPC, reinforcement learning, MARL, adaptive control policy learning, or Agave migration. Those remain later or separate research branches.
